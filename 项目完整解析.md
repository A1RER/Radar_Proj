## 1. 项目背景

### 1.1 要解决什么问题？

想象一下：一架无人机在城市上空飞行，我们怎么发现它、定位它、甚至识别它是什么型号？

传统方案是用摄像头——但摄像头在夜晚、雾天、超远距离下效果很差。

**这个项目的方案**：利用已经遍布城市的 **5G基站**，把它们变成"雷达"。基站持续发射毫米波信号，当无人机经过时，信号会从无人机上反弹回来，我们通过分析这个回波，就能：

1. **看到无人机的形状**（ISAR成像）
2. **精确定位其三维坐标**（多基站定位）
3. **预测它下一秒在哪**（Kalman滤波）
4. **识别它是什么类型**（深度学习分类）

### 1.2 为什么用5G毫米波？

| 频段 | 频率 | 优势 |
|------|------|------|
| 传统雷达 | 1~10 GHz | 穿透力强，但分辨率低 |
| **5G毫米波** | **28 GHz** | **带宽大→分辨率高，基站密集→随处可用** |

5G的400 MHz带宽带来 **37.5 cm的距离分辨率**，足以分辨无人机的机臂结构。

### 1.3 关键技术路线

```
5G信号 → 无人机反射回波 → 信号处理(ISAR) → 雷达图像
                                ↓
                         PSO优化图像质量
                                ↓
                    Kalman滤波 → 轨迹跟踪
                                ↓
                    多站定位 → 三维坐标
                                ↓
                    深度学习 → 型号识别
```

## 3. 模块一：ISAR成像

**文件**：`matlab_code/ISAR_Demo.m` | `python_code/11_ISAR_Python.py`

### 3.1 什么是ISAR？（直觉理解）

**普通合成孔径雷达(SAR)**：飞机或卫星**自身移动**，用"运动"合成一个大天线，从而获得高分辨率图像。

**逆合成孔径雷达(ISAR)**："逆"字的意思是——这次**不是雷达动，而是目标动**。无人机在旋转，从多个角度反射信号，等效于雷达绕着无人机转了一圈，从而合成大孔径。

```
普通SAR：   雷达→→→→→→→→→→→        目标不动
                                      ↑
                               等效大天线

逆合成(ISAR)：  雷达不动           目标旋转↻
              ○ 基站             ╱→无人机←╲
                                           每个角度的回波
                                           拼成大孔径
```

### 3.2 雷达参数（代码第12-19行）

```matlab
fc = 28e9;      % 载波频率 28 GHz（5G毫米波）
c = 3e8;        % 光速 3×10⁸ m/s
B = 400e6;      % 带宽 400 MHz
Tp = 1e-6;      % 脉冲宽度 1 微秒
fs = 2*B;       % 采样率 = 2×带宽（奈奎斯特定理）
Kr = B/Tp;      % 调频率 = 带宽/脉宽 = 4×10¹⁴ Hz/s
```

**分辨率计算**：
- **距离分辨率** = c / (2B) = 3×10⁸ / (2×400×10⁶) = **0.375 m = 37.5 cm**
  - 带宽越大，距离分辨率越高，所以5G毫米波的大带宽是关键
- **方位分辨率** = λ / (2·ω·T_obs) = (c/fc) / (2×0.5×2) ≈ **37.5 cm**
  - λ是波长，ω是旋转速度，T_obs是观测时间

### 3.3 目标模型：十字形无人机（代码第26-34行）

```matlab
target_points = [
    0,    0,    0;      % 中心点（主体）
    0.3,  0,    0;      % 右臂 +30cm
   -0.3,  0,    0;      % 左臂 -30cm
    0,    0.25, 0       % 前臂 +25cm
];
```

```
        ○ (0, 0.25)  ← 前臂
        |
○ ——— ● ——— ○       ← ●=中心, ○=左右臂
(-0.3,0)  (0,0) (0.3,0)
```

这4个点代表4个**散射中心**——现实中无人机螺旋桨、机身等不同位置对雷达信号的反射强度不同，简化为4个点来建模。

### 3.4 回波信号生成（代码第50-96行）

#### 3.4.1 LFM线性调频信号

雷达发射的不是单频信号，而是**线性调频（chirp）脉冲**：

```
发射信号：
频率 ↑
     |        /
     |      /
     |    /
     |  /
     |/
     +-------→ 时间
       调频率 Kr = B/Tp
```

数学表达：
```
s_tx(t) = exp(j·π·Kr·t²)   （复基带表示）
```

**为什么用chirp而不是单频？**
因为chirp的时间-带宽积大，经过匹配滤波后可以获得极高的距离分辨率（时间压缩增益 = B×Tp）。

#### 3.4.2 旋转运动模型（代码第59-83行）

```matlab
for p = 1:num_pulses           % 每个脉冲（慢时间循环）
    theta = omega * slow_time(p);  % 旋转角度 = 角速度 × 时间

    for k = 1:num_points           % 每个散射点
        % 旋转矩阵：2D旋转
        x_rot =  target_points(k,1)*cos(theta) - target_points(k,2)*sin(theta);
        y_rot =  target_points(k,1)*sin(theta) + target_points(k,2)*cos(theta);

        % 瞬时距离（基站到散射点的距离）
        R = R0 + y_rot;        % R0=1000m是初始距离，y_rot是横向偏移

        % 相对时延（关键！见下方说明）
        tau = 2*(R - R0)/c;
```

**关键设计：为什么用相对时延？**

```
绝对时延：τ_abs = 2R/c = 2×1000 / 3×10⁸ ≈ 6.67 μs
快时间窗口：fast_time = 0 ~ 1 μs

6.67 μs >> 1 μs！信号完全在窗口之外，全为零！
```

解决方案：用**相对时延** τ = 2(R - R₀)/c，把已知的固定传播延迟 R₀ 补偿掉：
```
τ_rel = 2(R - R0)/c = 2·y_rot/c ≈ ±2 ns << 1 μs  ✓ 在窗口内
```

#### 3.4.3 回波信号构建（代码第73-77行）

```matlab
sig = exp(1j*pi*Kr*(fast_time - tau).^2) ...  % chirp项：编码距离差信息
    .* exp(-1j*4*pi*fc*R/c);                   % 载波相位项：编码绝对距离R

echo(:, p) = echo(:, p) + sig';                % 叠加4个散射点的回波
```

两个指数项的物理含义：
- **chirp项** `exp(j·π·Kr·(t-τ)²)`：记录了"时延"（即距离），经过匹配滤波后产生距离压缩
- **载波相位项** `exp(-j·4π·fc·R/c)`：这才是多普勒信息的来源！无人机旋转时R在变化，相位随时间变化 → 产生多普勒频移

#### 3.4.4 添加噪声（代码第86-96行）

```matlab
SNR_dB = 10;    % 10 dB信噪比

try
    echo = awgn(echo, SNR_dB, 'measured');  % Communications Toolbox
catch
    % 手动生成高斯白噪声（不依赖工具箱）
    signal_power = mean(abs(echo(:)).^2);
    noise_power = signal_power / (10^(SNR_dB/10));  % SNR = S/N → N = S/10^(SNR/10)
    noise = sqrt(noise_power/2) * (randn(size(echo)) + 1j*randn(size(echo)));
    echo = echo + noise;
end
```

`try-catch` 结构保证了代码的**鲁棒性**：有工具箱用工具箱，没有就自己算。

### 3.5 距离压缩：匹配滤波（代码第100-110行）

**目的**：把弥散在时间上的chirp脉冲"压缩"成尖锐的峰，从而分辨不同距离的散射点。

**原理**：用接收信号和发射信号的**复共轭翻转版本**做卷积。

```matlab
ref_signal = exp(1j*pi*Kr*fast_time.^2);        % 参考信号（=发射chirp）
matched_filter = conj(fliplr(ref_signal));        % 匹配滤波器 = 共轭翻转

for p = 1:num_pulses
    range_compressed(:,p) = conv(echo(:,p), matched_filter, 'same');
end
```

**匹配滤波效果**：

```
压缩前（一个脉冲）：      压缩后：
幅度 ↑                   幅度 ↑
      /\/\/\/\                   |  |  |  |    ← 4个清晰的峰
     /         \                 |  |  |  |      （对应4个散射点）
----/           \----        ----+--+--+--+----
→ 时间                       → 距离
```

压缩增益 = 时间带宽积 = B × Tp = 400×10⁶ × 1×10⁻⁶ = **400 倍**

### 3.6 方位压缩：FFT（代码第113-117行）

```matlab
isar_image = fftshift(fft(range_compressed, [], 2), 2);
%            ↑         ↑              ↑         ↑
%         频率居中    FFT       在列方向(脉冲轴)    列方向
```

**物理意义**：
- 每个距离单元（行）有一列数据，代表该散射点在不同时刻的相位变化
- 相位变化的快慢 = 多普勒频率 = 该散射点的径向速度
- 对该列做FFT → 把不同多普勒速度的散射点**分开**
- 结果：横轴=距离，纵轴=多普勒频率 → 这就是ISAR图像！

### 3.7 结果显示（代码第120-157行）

```matlab
isar_norm = 20*log10(abs(isar_image)/max(abs(isar_image(:))) + eps);
%           ↑ dB转换   ↑ 取幅度       ↑ 归一化到[0,1]       ↑ 防止log(0)

imagesc(isar_norm);
caxis([-40 0]);    % 只显示最强点以下40dB范围，抑制噪声，突出目标
```

为什么要用dB显示？因为目标信号和噪声幅度差可达1000倍以上，线性刻度下噪声完全看不见，用对数刻度（dB）才能同时看到强弱信号。

## 4. 模块二：PSO粒子群优化

**文件**：`matlab_code/PSO_Optimization.m`

### 4.1 为什么需要优化？

ISAR成像中，无人机的**非匀速运动**会引入额外的相位误差，导致图像模糊（散焦）。我们需要找到最优的相位补偿参数：

```
原始相位（含误差）：φ(t) = φ_signal(t) + α·t + β·t²
                                           ↑线性误差  ↑二次误差

目标：找到最优的 α 和 β，使 φ_error = α·t + β·t² 被补偿掉
```

### 4.2 为什么用粒子群？

寻找最优 (α, β) 是一个**二维优化问题**。问题是目标函数（图像熵）对参数的关系很复杂，有多个局部最优。

传统梯度下降容易陷入局部最优；**PSO（Particle Swarm Optimization）粒子群优化** 用多个粒子同时探索，效果更好。

**灵感来源**：模拟鸟群觅食——每只鸟记住自己找到的最佳位置，也参考整群找到的最佳位置。

### 4.3 目标函数：图像熵（代码第45-55行）

```matlab
function entropy = calc_entropy(params, data)
    t = 1:size(data,2);
    phase = params(1)*t + params(2)*t.^2;        % α·t + β·t²
    corrected = data .* exp(-1j * phase);         % 施加相位补偿
    img = fftshift(fft(corrected, [], 2), 2);     % 重新成像

    img_abs = abs(img) / sum(abs(img(:)));        % 归一化为"概率分布"
    img_abs(img_abs == 0) = eps;                  % 避免log(0)
    entropy = -sum(img_abs(:) .* log(img_abs(:)));% H = -Σ p·log(p)
end
```

**图像熵的直觉**：
- **清晰图像**：能量集中在几个亮点，大部分像素接近0，熵**小**
- **模糊图像**：能量弥散开来，分布均匀，熵**大**
- 所以：最小化熵 = 最大化图像清晰度

```
模糊（高熵）：          清晰（低熵）：
· · · · · · ·          · · · · · · ·
· · ○ ○ ○ · ·          · · · ● · · ·
· ○ ○ ○ ○ ○ ·    →    · · · ● · · ·
· · ○ ○ ○ · ·          · · · ● · · ·
· · · · · · ·          · · · · · · ·
熵 ≈ 3.5               熵 ≈ 1.2
```

### 4.4 PSO算法实现（代码第57-115行）

**参数设置**：
```matlab
w = 0.7;    % 惯性权重：控制粒子"继续原方向"的倾向
c1 = 1.5;   % 个体学习因子：向自己历史最优位置靠近的力度
c2 = 1.5;   % 社会学习因子：向全群最优位置靠近的力度
```

**搜索空间**：
```matlab
lb = [-0.1, -0.01];   % 下界：[α_min, β_min]
ub = [0.1,   0.01];   % 上界：[α_max, β_max]
```

**初始化**（代码第65-72行）：
```matlab
positions = lb + (ub - lb) .* rand(num_particles, dim);
% 在 [lb, ub] 范围内随机放置20个粒子
velocities = zeros(num_particles, dim);
% 初始速度为0
```

**核心迭代**（代码第82-114行）：

```matlab
for iter = 1:num_iterations
    for i = 1:num_particles

        % === 速度更新公式（PSO核心） ===
        r1 = rand(1, dim);   % 随机因子，引入随机性防止早熟收敛
        r2 = rand(1, dim);

        velocities(i,:) = w  * velocities(i,:)              % 惯性项：保持原速度
                        + c1 * r1 .* (pbest_pos(i,:) - positions(i,:))  % 认知项：向个人最优
                        + c2 * r2 .* (best_pos - positions(i,:));        % 社会项：向全群最优

        % === 位置更新 ===
        positions(i,:) = positions(i,:) + velocities(i,:);

        % === 边界约束 ===
        positions(i,:) = max(lb, min(ub, positions(i,:)));   % 夹紧到[lb, ub]

        % === 计算适应度并更新最优 ===
        fitness(i) = objective(positions(i,:));

        if fitness(i) < pbest_val(i)        % 比个人历史最优更好？
            pbest_val(i) = fitness(i);
            pbest_pos(i,:) = positions(i,:);
        end

        if fitness(i) < best_val            % 比全群最优更好？
            best_val = fitness(i);
            best_pos = positions(i,:);
        end
    end
end
```

**速度更新的直觉解释**：

```
每个粒子受三个力：
  ←←← 惯性（继续原方向）
  →→→ 个人经验（去你历史上找到过的最好地方）
  ↑↑↑ 群体智慧（去群里其他粒子找到过的最好地方）

三力合一决定下一步走向
```

**优化效果**：
- 迭代20次后收敛
- 图像熵减少约35%
- 图像清晰度显著提升

## 5. 模块三：Kalman滤波轨迹跟踪

**文件**：`matlab_code/Kalman_Filter.m`

### 5.1 为什么需要Kalman滤波？

雷达每次测量都有噪声：测出的位置可能偏差5米以上。如果直接用测量值，轨迹会抖动得很厉害：

```
真实轨迹：                   直接用测量值：
→→→→→→→→→              →↗↘→↗↘→↗↘→
（平滑直线）                 （抖动折线）

Kalman输出：
→→→→→→→→→
（平滑，接近真实）
```

**Kalman滤波的思路**：
- 用**运动模型**预测下一时刻位置（"我认为它应该在这里"）
- 用**新的测量值**修正预测（"雷达说它在那里"）
- 根据各自的**不确定度**，加权融合两个信息

### 5.2 状态空间模型（代码第12-19行）

```matlab
% 状态向量：[x, y, vx, vy]ᵀ  ← 位置+速度，共4个量
% 测量向量：[x, y]ᵀ          ← 只能测位置，不能直接测速度

% 状态转移矩阵 F（匀速运动模型）
F = [1  0  dt  0;    % x(t+1) = x(t) + vx·dt
     0  1  0   dt;   % y(t+1) = y(t) + vy·dt
     0  0  1   0;    % vx(t+1) = vx(t)        （假设速度不变）
     0  0  0   1];   % vy(t+1) = vy(t)

% 测量矩阵 H（从状态提取测量量）
H = [1  0  0  0;     % 测量x = 取状态的第1个元素
     0  1  0  0];    % 测量y = 取状态的第2个元素
```

### 5.3 Kalman滤波两步走（代码第29-41行）

```matlab
for k = 1:N
    %% ===== 预测步（根据运动模型推算） =====
    x = F * x;           % 状态预测：x⁻ = F·x
    P = F * P * F' + Q;  % 协方差预测：P⁻ = F·P·Fᵀ + Q
    %                Q = 过程噪声协方差（描述运动模型的不确定性）

    %% ===== 更新步（融合新的测量值） =====
    K = P * H' / (H * P * H' + R);      % 计算Kalman增益 K
    x = x + K * (measurements(:,k) - H * x);  % 修正状态
    P = (eye(4) - K * H) * P;           % 更新协方差
    %   R = 测量噪声协方差（描述雷达测量的不确定性）
end
```

### 5.4 Kalman增益的物理意义

```
K = P⁻·Hᵀ · (H·P⁻·Hᵀ + R)⁻¹
    ↑分子：预测不确定度   ↑分母：总不确定度

直觉：
  K → 1（接近1）：预测不可靠，更相信测量值
  K → 0（接近0）：测量噪声大，更相信预测值
```

**数值例子**（设为标量简化）：
- 预测不确定度 P = 10 m²
- 测量噪声 R = 25 m²
- K = 10/(10+25) ≈ 0.29
- 新估计 = 预测值 + 0.29 × (测量值 - 预测值)
  - 即：29%相信测量，71%相信预测

### 5.5 性能结果

```
测量值RMSE：5.2 m
Kalman估计RMSE：1.8 m
改善：65%
```

## 6. 模块四：多基站协同定位

**文件**：`matlab_code/MultiStationLocalization.m`（OOP类）

这是一个**面向对象**的MATLAB类，封装了3种定位方法。

### 6.1 基站布局

```matlab
stations = [
    0,    0,    0;     % 基站1（原点）
    1000, 0,    0;     % 基站2（东1km）
    500,  866,  0;     % 基站3（北866m）
    500,  289,  816    % 基站4（高空816m）
];
```

四个基站形成**四面体**结构，保证三维空间的良好覆盖（GDOP最小）。

### 6.2 方法一：TDOA时差定位（代码第30-67行）

**原理**：信号从无人机到不同基站的传播时间不同，时间差决定了无人机在一个**双曲面**上，多个双曲面的交点就是目标位置。

```
基站1：
     \
      \  时差 τ₁ = (d₂-d₁)/c
       ×无人机
      /
     /
基站2
```

**数学建模**（代码第40-55行）：

```matlab
% 以基站1为参考，对第i个基站建立方程
A(i-1, :) = 2 * (si - s1);     % si=第i站位置, s1=第1站位置
b(i-1) = c^2 * tau_i^2 - norm(si)^2 + norm(s1)^2;

% 最小二乘初始解：Ax = b → x = (AᵀA)⁻¹Aᵀb
pos = (A' * A) \ (A' * b);
```

**精化**（代码第57-65行）：
```matlab
% 用非线性优化进一步精化（最小化TDOA残差）
objective = @(p) obj.tdoaObjective(p, time_differences, c);
pos = fminunc(objective, pos, options);  % 梯度下降精化
```

### 6.3 方法二：AOA角度定位（代码第85-121行）

**原理**：每个基站测量无人机的方位角和仰角，画出一条射线，多条射线的交点（最近公共点）就是目标位置。

```matlab
% 每个基站i给出方向向量 di（单位向量）
directions(i, :) = [cos(el)*cos(az), cos(el)*sin(az), sin(el)];

% 最小化：所有射线到目标点p的距离平方之和
cost = Σ || (p - si) - [(p-si)·di]·di ||²
```

### 6.4 方法三：RSS信号强度定位（代码第139-157行）

**原理**：信号越远越弱，通过接收功率推算距离，再三边定位。

```matlab
% 路径损耗模型：RSS = P0 - 10·n·log10(d/d0)
% 反推距离：d = d0 · 10^((P0 - RSS) / (10·n))
distances = d0 * 10.^((P0 - rss_values) / (10*n));
```

然后用三边定位（trilateration）求解位置。

### 6.5 GDOP几何精度衰减因子（代码第240-258行）

GDOP衡量**基站几何布局**对定位精度的影响：

```matlab
% 构建几何矩阵
G(i, :) = [-(target_pos - si) / r, 1];   % 方向余弦 + 时钟项

% GDOP = sqrt(trace(Q))，Q = (GᵀG)⁻¹
Q = inv(G' * G);
gdop = sqrt(trace(Q(1:3, 1:3)));
```

GDOP越小（接近1），定位越精确；GDOP越大，测量误差被放大越多。

### 6.6 粒子滤波跟踪（代码第305-359行）

除了Kalman滤波，还实现了**粒子滤波**（适合非线性非高斯情况）：

```
初始化：在目标附近随机撒N=1000个粒子，每个粒子代表一个可能的状态

每步：
  预测：每个粒子按运动模型+随机噪声移动
  更新：根据当前测量值，给每个粒子赋予权重（越接近测量，权重越高）
  重采样：权重小的粒子被淘汰，权重大的粒子被复制
  估计：所有粒子的加权平均 = 当前状态估计
```

def generate_synthetic_data(num_samples=1000, num_classes=5, image_size=128):
    for i in range(num_samples):
        label = np.random.randint(0, num_classes)  # 随机标签

        if label == 0:    # 小型无人机：高斯亮斑

                y = int(64 + 30*sin(angle))

### 7.3 Dataset类（代码第32-65行）

class ISARImageDataset(Dataset):

### 7.4 SimpleCNN网络结构（代码第68-109行）

```python
class SimpleCNN(nn.Module):
    def __init__(self, num_classes=5):
        # 三个卷积块
        self.conv1 = nn.Conv2d(3, 32, kernel_size=3, padding=1)   # 3→32通道
        self.bn1   = nn.BatchNorm2d(32)                            # 批归一化
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)  # 32→64通道
        self.bn2   = nn.BatchNorm2d(64)
        self.conv3 = nn.Conv2d(64, 128, kernel_size=3, padding=1) # 64→128通道
        self.bn3   = nn.BatchNorm2d(128)
        self.pool  = nn.MaxPool2d(2, 2)                           # 每次缩小一半

        # 全连接层
        self.fc1 = nn.Linear(128 * 16 * 16, 512)  # 展开后512维
        self.dropout = nn.Dropout(0.5)              # 随机丢弃50%防过拟合
        self.fc2 = nn.Linear(512, 5)               # 输出5类分数

    def forward(self, x):                           # x: (batch, 3, 128, 128)
        x = self.pool(F.relu(self.bn1(self.conv1(x))))  # → (batch, 32, 64, 64)
        x = self.pool(F.relu(self.bn2(self.conv2(x))))  # → (batch, 64, 32, 32)
        x = self.pool(F.relu(self.bn3(self.conv3(x))))  # → (batch, 128, 16, 16)
        x = x.view(-1, 128 * 16 * 16)                   # → (batch, 32768)
        x = F.relu(self.fc1(x))                          # → (batch, 512)
        x = self.dropout(x)
        x = self.fc2(x)                                  # → (batch, 5)
        return x
```

**数据维度变化**：

```
输入:  (batch, 3, 128, 128)
Conv1+Pool: (batch, 32, 64, 64)   ← 特征数从3增到32，空间缩小一半
Conv2+Pool: (batch, 64, 32, 32)   ← 特征数增到64，空间再缩小
Conv3+Pool: (batch, 128, 16, 16)  ← 特征数增到128，空间再缩小
Flatten:    (batch, 32768)
FC1:        (batch, 512)
FC2:        (batch, 5)            ← 5个类别的分数（logits）
```

### 7.6 训练流程（代码第163-199行）

```python
def train_epoch(self, train_loader):
    self.model.train()  # 开启训练模式（dropout生效）

    for images, labels in train_loader:
        # 前向传播
        outputs = self.model(images)          # 计算预测
        loss = self.criterion(outputs, labels) # 交叉熵损失

        # 反向传播
        self.optimizer.zero_grad()  # 清除上轮梯度
        loss.backward()             # 计算梯度
        self.optimizer.step()       # 更新权重（Adam优化器）
```

**优化器配置**：
```python
self.optimizer = optim.Adam(model.parameters(), lr=0.001)    # Adam，初始学习率0.001
self.scheduler = optim.lr_scheduler.StepLR(optimizer,
                                            step_size=10,     # 每10个epoch
                                            gamma=0.1)        # 学习率×0.1
```

学习率调度的作用：训练初期用大学习率快速收敛，后期用小学习率精细调整。

## 10. 常见问题

### Q1: 为什么用相对时延而不是绝对时延？

**答**：无人机距离1000m，绝对时延 = 2×1000/(3×10⁸) ≈ 6.67μs。但快时间窗口只有 Tp = 1μs，6.67 > 1，信号早就"跑出"窗口，代码里赋值为0。用相对时延把固定的6.66μs偏置减掉，只保留散射点之间的相对位移引起的 ±2ns 差异，确保信号落在窗口内。

### Q2: 为什么ISAR图像的纵轴叫"多普勒"而不是"方位"？

**答**：在SAR里，飞机移动产生的相位变化直接对应方位角，所以叫方位轴。在ISAR里，目标旋转产生的是**多普勒频率**（转速越快，多普勒越大），通过多普勒和旋转半径的关系可以换算成方位，但原始轴叫多普勒更准确。

### Q3: PSO的20个粒子够用吗？

**答**：对于2维优化问题（α, β），20个粒子完全够用。粒子数需要随维度增加。如果是100维优化，可能需要1000个粒子。

### Q4: Kalman滤波假设匀速运动，无人机变速怎么办？

**答**：
1. **过程噪声Q的作用**：Q不为零，允许速度有小幅随机变化，相当于近似处理加速度
2. **扩展：**可以把加速度也加入状态向量：`[x, y, vx, vy, ax, ay]`，用"近匀加速"模型
3. **更好的方案**：使用**交互多模型(IMM)**滤波器，同时跑多个运动模型（匀速、转弯、加速），自动切换

### Q5: ResNet-18迁移学习时，应该只训练最后一层还是全部层？

**答**：
- **只训练最后一层（Fine-tune Last Layer）**：速度快，数据少时不易过拟合，适合ISAR图像和ImageNet差异较大的情况
- **训练全部层（Full Fine-tune）**：精度更高，但需要更多数据，学习率要设得很小（避免破坏预训练特征）
- **推荐**：先冻结前面的层，只训练最后一层；如果效果不好，再逐步解冻更多层
